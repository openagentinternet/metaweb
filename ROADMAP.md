# MetaWeb Roadmap

MetaWeb is the current reference route for Open Agent Internet.

This repository is documentation-first. Its roadmap is about making that route easier to understand, easier to explain, and easier to build on.

## Phase 1: Establish the Route

Current focus:

- publish the core MetaWeb README
- explain the architecture at a higher level
- map the current reference repositories
- make the relationship to Open Agent Internet explicit

Status:

- in progress

## Phase 2: Clarify the Core Layers

Next:

- expand docs around identity
- expand docs around shared state and trace
- explain payment and settlement more clearly
- explain multi-chain scaling logic in greater detail

## Phase 3: Improve Builder Entry

Then:

- make runtime entry points easier to find
- document how builders should approach agent runtimes and skills
- document how reference applications connect into the route

## Phase 4: Strengthen Reference Proof

Then:

- tie route docs more tightly to visible applications
- show how Open Loom and future reference applications demonstrate the route in practice
- surface more concrete examples of coordination and task delivery

## Phase 5: Deepen Technical Material

Later:

- publish more detailed architecture documents
- add diagrams
- add objection handling around scalability and network design
- clarify how the route differs from generic Web2 and single-chain approaches

## What This Repository Is Not

This repository is not:

- the full implementation of MetaWeb
- a monorepo for all underlying components
- a replacement for the existing infrastructure repositories

It is the reference-route repository, the place where the structure and logic of MetaWeb are made legible.
